Overview

Learn how to use the HBZ-Forum API with all its potential! You can use it in order to acces users, articles, policies and much more. If you have an account you can also utilize it in order to do all of the account-onyl stuff like liking articles, writing articles, comments or automatically chatting / creating chatbots!

Structure

There are generally 2 API endpoints availible: /forum/assets/api/ and its subdirectories as well as /forum/v2/assets/api/. They both contain different endpoints with different abilities.

The whole system of this website is built upon Ids: every instance, be it users, articles, comments, views or likes are given ids. If you want to edit or remove something you will have to know its Id. To get it you can use the numerous data endpoints availible.

Most API endpoints will return JSON objects with the http data type application/json. Below you can see an example output:

              
                  {
                    “likeId“: 132, 
                    “userId“: 32,
                    “targetUserId”: 343,
                    “likeCreated“: 9238263424
                  }

Here are a few things to note:
- Times are mostly returned as UNIX-Timestamps. You can then convert those into human readable formats.
- If the action requires a userId, it is returned as userId and then if it affects other users those are characterized by targetUserId. If it only affects a user, the user is given as userId.
- This API is not ratelimited. Abuse will, however, be punished and the responsible addresses permanently blocked.

Issues & FAQ

This API was mainly made as a way to communicate between front- and backend. It is, therefore, not optimized for usability or ease-of-use. If you have any problems with using the endpoints you could first try to analyze the webtraffick when issuing the same action on the webpage. If it still doesnt work as expected/intended, you can contact us using a Report or by writing an Email. You can see how to contact us in the impressum of the webpage.

If you find any bugs/glitches/mistakes or have other remarks, please also dont be too shy to contact us as described on top.
