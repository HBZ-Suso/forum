Introduction

First of all, thank you very much for using this webpage!

This documentation is to help you regarding the API of the Forum of the Highly Skilled class of the Heinrich Suso Gymnasium.

Requirements

You will need the following sofwares to interact with the API

  1. Code Editing Software (eg: Dreamweaver, Sublime Text or Notepad) if you want to write programms
  2. Web Browser for testing and easy access (eg: Google Chrome or Mozilla Firefox)
  3. Some understanding of Webdevelopement

Getting Started

This webpage including the API is coded with PHP and MySQL for the database. The API endpoints can be accessed in the subdirectory /forum/assets/api/. For a complete list of all endpoints see Usage.

Using this Documentation:

This documentation was created for you to be able to understand how the API of the forum works. It aims to explain the endpoints and their usage.

Limits

Ratelimits

There is no direct ratelimiting on the API, but it does have a access logger and block functionaltiy. Thus if you overuse this API or abuse it, you will get IP-blocked.

Surveillance

We track the amount of requests and their details in order to provide a better experience and protect the functionality of the API. For more details visit the policy page.

Usage

Information

This is a list of all provided endpoints. Some of them may require accounts to use.

change_article.php

  • This endpoint enables users to change articles. Requires you to own the article or be administrator.

    Arguments

    articleId - Id of the article

    articleData - json array, containing either articleText or articleTitle. The endpoint changes those to the entered values.

change_article.php

  • This endpoint enables users to change their account data. Requires you to be logged in.

    Arguments

    change_data - json array, containing userPassword, userMail, userAge, userPhone, userDescription. Password and Mail connot be used with linkauths. The endpoint changes those to the entered values.

comment.php

  • Enables you to comment under article or user. Requires you to be logged in.

    Arguments

    articleId - Id of the article

    userId - Id of the user

create_report.php

  • Create report

    Arguments

    title - title of the report

    text - text of the report

delete_comment.php

  • Enables you to delete a comment under an article or user. Requires you to be logged in.

    Arguments

    articleId - Id of the article

    userId - Id of the use.

    commentId - Id of the comment

delete.php

  • Deletes user or article. Deleting users is not allowed with linkauth

    Arguments

    article - Id of the article

    userId - Id of the user.

get_article_list.php

  • This endpoint provides a JSON array of all articles.

    Arguments

    search - provide a string that is searched for

    max - provide the maximum amount of articles returned (sorted by date). Capped at 1000.

get_article.php

  • This endpoint provides a JSON array of the searched article.

    Arguments

    articleId - provide the articleId to be returned

    articleTitle - provide the articleTitle to be returned

get_articleHTML.php

  • This endpoint provides a html site display of the searched article.

    Arguments

    articleId - provide the articleId to be returned

    articleTitle - provide the articleTitle to be returned

get_articles.php

  • This endpoint provides a html site display of the article block. It returnes the full html list block. It uses the page session values.

    Arguments

    search - able to enter search string

get_chosen_section.php

  • This returns the chosen section. If you have a cookies allowed, one will also be used by the wegpage.

get_comments.php

  • This endpoint provides a json array of the comments on chosen article or user.

    Arguments

    articleId - Id of the article

    userId - Id of the user

get_current_username.php

  • If logged in, returns the username.

get_error_argument.php

  • Returns a json array of infos for the provided error code.

    Arguments

    error_argument - code of the error

get_highlights.php

  • This endpoint provides a html site display of the highlight block. It returnes the full html list block. It uses the page session values.

get_language.php

  • Returns the selected language

get_languages.php

  • Returns a json array of all availible languages

get_link_user.php

  • Returns the user a link is connected to.

    Arguments

    linkauth - link authentification code

get_themes.php

  • Returns a json array of all availible themes.

get_user_list.php

  • Returns a json array of all users.

    Arguments

    search - search string

    max - max amount of users to return, limited to 1000, default 100

get_user.php

  • Returns json data of the user.

    Arguments

    userId - Id of the user

    userName - Name of the user

get_userHTML.php

  • Returns the html of an user page.

    Arguments

    userId - Id of the user

    userName - Name of the user

get_users.php

  • Returns the html of an userblock page. Uses the page session values.

    Arguments

    search - search string

language.php

  • Sets the language.

    Arguments

    language - language to set to

like.php

  • Likes user or article.

    Arguments

    userId - Id of the user

    articleId - Id of the article

link_valid.php

  • Returns valid or invalid if link authentification code is avlid or invalid.

    Arguments

    linkauth - link authentification code

link.php

  • Creates an authentification code for the selected user and returns it.

    Arguments

    username - name of the user

    password - password of the user

    linkinfo - info provided for the link

lock.php

  • Locks a users account. Requires moderator or administrator.

    Arguments

    userId - Id of the user

login_with_link.php

  • Logs in with a link authentification code.

    Arguments

    linkauth - link authentification code

login.php

  • Log into account.

    Arguments

    username - name of the user

    password - password of the user

logout.php

  • Logs out.

set_articlePage.php

  • Sets the current article Page.

    Arguments

    articlePage - Page to set to

set_chosen_section.php

  • Sets the current section.

    Arguments

    section - section to set to

set_highlightPage.php

  • Sets the current highlight Page.

    Arguments

    highlightPage - Page to set to

set_resolution.php

  • Sets the saved resolution. Used to determine if a mobile view is needed.

    Arguments

    res_x - x resolution

    res_y - y resolution

set_userPage.php

  • Sets the current user Page.

    Arguments

    userPage - Page to set to

theme.php

  • sets the current theme.

    Arguments

    theme - Theme to set to

Verify.php

  • Verifies user. Needs administrator.

    Arguments

    userId - user to verify
